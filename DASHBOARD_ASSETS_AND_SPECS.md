## B. DeepSeek Prompt 骨架

> 铁律(每个 prompt 都要体现):① 只呈现事实/做分类,**不下投资判断** ② 强制行内引用,无来源不写 ③ 分类做选择题 ④ 输出严格 JSON,便于解析。

### B1. 新闻结构化(选择题式,批量)

```
你是金融信息整理器,不是分析师。对以下新闻做结构化,严禁编造原文没有的信息。

【新闻原文】
{title}\n{body}\n来源:{source} 时间:{time_utc8}

【任务】只做分类和提取,输出JSON:
{
  "themes": [从固定清单选,可多选,没有则空数组],   // 清单见 theme_node_map.json 的键
  "tickers": [原文明确提到的股票名/代码,原文没有就空],
  "sentiment": "利好|利空|中性|澄清",              // 四选一
  "event_type": "公告|政策|涨跌异动|研报|互动易|外盘|其他",
  "one_line": "用原文原话概括一句,不加解读,≤40字",
  "source_url": "{url}"
}
规则:themes 只能从给定清单选词,不许自创;one_line 不许出现"利好X""看好Y"等判断;
tickers 只填原文明确出现的,不许推断。
```

### B2. 新闻→节点翻译(规则优先,LLM兜底)

**优先用规则:** 拿 B1 输出的 `themes`,查 `theme_node_map.json` 得到 node_id 列表,再查 `stock_node_map.json` 得到关联股票。**规则命中就不调 LLM。**

**仅当规则未命中且新闻明显相关时,用 LLM 做选择题:**
```
以下新闻规则未匹配到产业链节点。从【节点清单】中选出最相关的(可多选,无相关选"NONE")。
只做匹配,不解释,输出JSON: {"node_ids": [...]}
【节点清单】{nodes.json 的 node_id 列表}
【新闻】{one_line}
```

### B3. 每日报告生成(核心,分盘前/盘后两套)

**通用系统提示:**
```
你是投研信息整理器,为一位每日做决策的A股AI科技投资者服务。
你的职责:呈现"今天发生了什么变化",不做投资判断。
铁律:
- 每个事实必须带来源(标注 [来源] )。没有来源的信息一律不写。
- 主线那栏只做"基于数据的中性事实陈述"(如"资金从X净流出、流入Y"),
  绝不写"所以该关注Z""看好W"这类倾向性结论——那一栏留 <使用者填写> 占位。
- 只用我提供的数据,不调用外部知识补充,不编造公告/数字。
- 数据截止时点必须标注(UTC+8)。
输出严格JSON,结构见下。
```

**盘前 user 提示:**
```
【数据截止 UTC+8】{ts}
【昨夜美股(链相关票)】{us_data}
【隔夜外盘/宏观】{macro}
【今晨盘前公告】{premarket_announcements}
【昨日A股盘后快照】{yesterday_snapshot}

输出JSON:
{
  "data_cutoff": "{ts} UTC+8",
  "session": "premarket",
  "headline": {"fact":"中性事实陈述","user_judgment":"<待填>","confidence":"高|中|低"},
  "top3": [
    {"change":"变化描述","evidence":"[来源]","node_ids":[...],"related_stocks":[...]},
    ...3条
  ],
  "sectors": [{"chain":"光通信","status":"一句状态[来源]"}, ...],
  "falsification": [
    {"claim":"某判断","condition":"证伪条件(具体可1-2周验证)","draft_by":"deepseek"}
  ]
}
```

**盘后 user 提示:** 同结构,`session":"afterhours"`,输入换成【今日A股盘面】【今日盘中公告/互动易】【昨夜美股(背景)】,top3 侧重"今天谁涨谁跌/资金往哪切/情绪冷热"。

> **证伪条件流程:** DeepSeek 起草 `falsification[].condition` → 前端展示时标"待审定" → 使用者可编辑 → 确认后钉死打时间戳存入账本(只读)。

## C. 前端视觉规格(机构级 / 对标 Bloomberg 终端原则)

> **设计标杆是 Bloomberg 终端,学的是原则不是外观:每个像素都要问责,层级由"重要性"决定而非装饰,数字要能瞬间读取。** timsun 那种个人站的松散排版是反面教材,不要参考它的视觉。
> 核心信条(机构金融UI铁律,有据可依):
> 1. **可读性压倒一切** —— 不把数字藏在"友好"可视化背后,不为亲和牺牲 legibility。
> 2. **红绿是信号不是装饰** —— 中性基底 + 高对比数据 + 精准克制的强调色。
> 3. **数据密度是优点** —— 一屏看到全局,能扫描大量行不疲劳,而非大量留白。
> 4. **克制 ≠ 死板** —— 强调色用在真正重要处,别做成灰蒙蒙没层级的"安全脸"。

### C0. 技术选型与全局设计系统

**技术栈:** 推荐 **React + TypeScript + Tailwind + shadcn/ui**(2026 机构级仪表盘事实标准,Fortress/Vault 等 Bloomberg 风模板均基于此)。图表用 **ECharts**(数据密集型金融图表能力强)或 lightweight-charts(K线级)。**虽然后端静态生成,前端仍可 React 构建后导出静态 bundle**,不冲突。若坚持极简可用原生JS+ECharts,但 shadcn/ui 省大量组件工作。

**设计令牌(CSS变量,dark-first):**
```
--bg-base:      #0A0E14   /* 主背景,近黑带蓝调,非纯黑(减少眩光) */
--bg-surface:   #131820   /* 卡片/面板 */
--bg-elevated:  #1C2430   /* 悬浮层/hover */
--border:       #232B36   /* 分隔线,极细 1px */
--text-primary: #E4E9F0   /* 主文字 */
--text-muted:   #8B95A5   /* 次要文字/标签 */
--text-dim:     #5A6474   /* 时间戳/元信息 */
--up:           #F6465D   /* A股涨=红(A股习惯,勿用西方绿涨) */
--down:         #2EBD85   /* A股跌=绿 */
--accent:       #F0B90B   /* 琥珀金,强调色,仅用于关键信号(Bloomberg风) */
--accent-info:  #4A9EFF   /* 信息蓝,链接/可点击 */
```
> **A股红涨绿跌是硬约束**,与美股/Bloomberg相反,染色务必按A股。美股数据区单独标注,或统一用箭头+符号不单纯依赖颜色(色盲友好,Bloomberg亦如此)。

**字体:** 数字/行情/代码用等宽(`Geist Mono`/`JetBrains Mono`/`ui-monospace`);中文正文用 PingFang SC / Microsoft YaHei / system-ui;英文 Inter。**所有数字必须等宽,保证竖排对齐可扫描 —— 机构表格命脉。**

**密度:** 高密度。表格行高 28–32px(非消费级48px+)。卡片 padding 12–16px。分隔用 1px 极细线,不堆阴影。圆角小(4px)。

**布局骨架:** 左侧固定窄导航栏(图标+文字,六模块)+ 顶部状态条(数据截止时点/市场状态/时钟 UTC+8)+ 主内容区。**用左侧栏,不用顶部横向大导航。**

**动效:** 极克制。数据更新用微妙高亮闪烁(flash后即褪,Bloomberg风),不用滑入弹跳。尊重 `prefers-reduced-motion`。

### C5. 通用组件
- KPI卡:标签(muted小)+数值(等宽大)+变化(色/箭头)。
- 表格:斑马纹极淡或无,行hover,表头sticky,数字右对齐等宽,可点排序。
- badge:小圆角,语义色,暗背景上低饱和不刺眼。
- 空状态:给指引("暂无盘中异动")不留白。加载用骨架屏不转圈。
- 来源链接:统一 info-blue+外链↗,可点回原文(呼应可追溯防幻觉)。

## D. 存储格式(已定)

- **主数据:SQLite**(单文件、静态站友好、无需数据库服务)。表:`stocks`, `nodes`, `stock_node`, `news`, `announcements`, `fund_flow`, `daily_report`, `ledger`(账本,只读约束), `fund_letters`, `research_reports`。
- **账本只读:** `ledger` 表插入后不允许 UPDATE/DELETE(用触发器或应用层强制),每行带 `created_at_utc8` 时间戳。
- **静态页生成:** 后端定时任务查 SQLite → 渲染成静态 HTML(可用 Jinja2 模板)→ 部署。前端 JS 只做客户端筛选/折叠,不发请求(或只读一份导出的 JSON)。
- **图谱:** 第一版用 `nodes.json` + `stock_node_map.json` 即可,不必上图数据库。关系边(§spec 8类)第一版可省略,后续需要再加 `edges` 表。
