> 本文档是给 Claude Code 的施工规格。所有决策已闭环。

## 0. 产品定位(不可动摇的北极星)

**一句话:** 每天早上/收盘后 5 分钟,让使用者比不看它多知道"今天该关注哪个环节/哪只票、为什么、以及什么情况证明看错了"。

**本质:** 决策雷达,不是参考图谱。**捕捉"变化/增量",不陈列"状态/存量"。**

**基调:** 牛市视角 —— 主维度是叙事强度 + 资金流向;扣非只做**显示**不做**筛选**(不砍任何票)。唯一保留的纪律是**证伪条件**。

**使用者画像:** 每天看盘、做短中线决策的独立量化/价值投资者。所以日报(盘前+盘后)频率是合理的。

**判断权铁律:** LLM(DeepSeek)只负责**呈现变化 + 提取事实 + 做分类(选择题)**,**绝不下投资判断**。主线判断留白给使用者填。原因:模型对 A 股产业链的理解不如使用者,让它下判断只会得到"流畅但平庸且诱导放松警惕"的结论。

## 4. 六模块规格

> **贯穿维度:持仓/自选层(见 SUPPLEMENT_2 §G)。** 每个模块都要读使用者的持仓/自选,做"优先高亮+优先排序+优先推送"。这是"跟我的钱相关的雷达"的核心,不是可选项。每日报告有"我的持仓动态"板块,新闻有"仅看我的票"开关,热力图持仓票描边,推送持仓异动最高优先级。

### 4.1 核心层

#### 模块①:每日报告(心脏)

**生成:** 全 DeepSeek。盘前 + 盘后各一次。

**时点切片表(UTC+8):**
```
盘前报告(约 08:30 生成):
  输入 = 昨夜美股收盘(链相关票) + 隔夜外盘/宏观 + 今晨盘前公告 + 昨日A股盘后快照(背景)
  产出 = "今天开盘该盯什么"
盘后报告(约 15:30 生成):
  输入 = 今日A股盘面(涨跌/资金/封板/异动) + 今日盘中公告/互动易 + 昨夜美股(背景)
  产出 = "今天发生了什么、明天怎么演绎"
```
**每份报告顶部强制标注数据截止时点(UTC+8),防时序错配。**

**四段式结构:**
1. **今日主线(1句+置信度)**:DeepSeek 只做**基于数据的中性事实陈述**(如"资金今日从光模块净流出、流入存储")。**带倾向的结论留白给使用者填。**
2. **今天只看这3件事**:每件 = 变化描述 + 证据(带来源链接) + 关联到175只票的哪个节点。
3. **分板块扫描(可折叠)**:光通信/半导体/算力配套/MLCC/机器人 各一句状态,背景板。
4. **证伪与风险**:每个判断挂证伪条件。**证伪条件由 DeepSeek 起草,使用者审定后钉死打时间戳(每天唯一必须人工介入的一步,约30秒)。**

**判断复盘账本(不是"胜率账本"):**
- 每条主线判断:内容 + 证据 + 证伪条件,生成即**打 UTC+8 时间戳、只读、不可编辑**(系统强制,不靠意志力)。
- 验证锚点 = **证伪条件是否触发**,不是"涨没涨"(牛市里啥都涨,涨跌是运气不是认知)。
- 被证伪时,DeepSeek 辅助归类错在哪一环:**信息错 / 逻辑错 / 纯运气**(三选一,选择题)。
- 产出 = **错误类型分布**(发现系统性偏差),不是一个诱发自欺的胜率分数。
- **定位:学习工具,不是审判工具。**
- **防注水:** 证伪条件由 DeepSeek 生成,使用者**每周复盘时抽查**几条是否写太松(如"除非整体崩盘"这类几乎不可能触发的),发现放水就在 prompt 加约束("证伪条件须具体、可在1-2周内验证")。

#### 模块②:新闻/事件(每日报告的原料供应商)

**管线:采集 → 规则漏斗 → 翻译层 → 输出**

- **采集层**:公告(Tushare/巨潮)、打板异动(东财四池+涨停原因+连板梯队)、资金(北向/龙虎榜/主力净流入)、互动易问答、外盘(yfinance)、RSS(华尔街见闻等)。
- **规则漏斗**:白名单过滤,只留命中175只票/8条链的,几千条压到几十条。**纯规则,不烧LLM。**
- **翻译层(核心差异化)**:把外部事件自动映射到"影响你哪个节点/哪只票"。如"黄仁勋称HBM供给紧"→"利好存储链兆易/香农芯创"。先用规则(题材→节点映射表),不够再上 DeepSeek。
- **输出**:黄金模板事件流 `[题材标签] + 股票 + 异动类型 + 原因(带来源日期)`(抄同花顺"异动股揭秘"格式);盘前/盘后各出一份快照喂每日报告。

**展示:** 按链节点分组的事件流 + 顶部**情绪温度计**(今日涨停X/炸板X/封板率X%/最高连板,抄财联社盘后四件套)。

### 4.2 支撑层

#### 模块③:AI热力图(状态背景板)

- **四象限**(抄 timsun):叙事强度 × 财报兑现 × 估值高低。节点用175只票表。
- 数据自动挂:Tushare 拉市值/涨跌/PE/PS/营收增速/毛利率。
- 冷启动需历史行情(见 §5)。

#### 模块④:半导体资讯

**不是独立模块**,是新闻层 `sector=半导体` 的一个筛选视图。

### 4.3 慢速层

#### 模块⑤:研究库(投行研报)

东财 reportapi / 同花顺自动拉个股+行业研报,DeepSeek 做结构化摘要(选择题式)。不手动录入。

#### 模块⑥:基金信函

**接海外对冲基金季度信聚合圈**(不自建信源清单,抄 timsun 做法)。AWS 台湾定时抓 → DeepSeek 中文摘要:3条核心观点 + 立场(看多/看空/谨慎/混合,四选一)+ 策略分类(六选一)+ **相关度评分(按"对你AI科技链的启发"打分,这是差异化)**。处理不过来标"待分类"占位。展示抄 timsun 卡片式,可按季度/策略/立场筛选。季度低频。

## 5. 采集频率与冷启动(已定)

**采集频率(UTC+8):**
- 盘中交易时段(09:30–15:00):**每小时整点抓一次**(公告/行情/资金/异动)
- 收盘后 15:30:拉一次完整盘后快照(给盘后报告)
- 盘前 08:30:拉隔夜外盘 + 晨间公告(给盘前报告)
- 非交易时段:不抓
- 遵守东财系接口限流(开源项目内置 `EM_MIN_INTERVAL` 串行限流,别调激进)
- **不上全天候实时**(对该决策频率是浪费 + 增加封IP和维护负担)

**冷启动(上线前一次性回灌):**
- 行情历史:**用 Tushare** 拉175只票+指数过去1年日线(几分钟)。`[开工现场看]` 阿里云DC是否已有历史数据;大概率直接用 token 拉最干净。
- 新闻/公告:不回灌(历史新闻价值低),从上线日起积累。
- 账本:天然从第一篇报告积累,无需历史。

## 6. 防幻觉纪律(所有 LLM 环节铁律,不可放松)

> 依据 2026 年 LLM 研究:金融高风险领域即便顶级模型+联网,幻觉率仍约30%。**喂什么数据比用什么模型重要十倍。**

1. **强制行内引用 grounding**:每句事实性陈述后挂 `[来源:链接/公告]`,无来源不许写。
2. **让 LLM 做选择题不做问答题**:题材归类给固定清单选,情绪给"利好/利空/中性/澄清"四选一,错误归因给"信息/逻辑/运气"三选一。不自由生成。
3. **DeepSeek 只呈现事实/做分类,不下投资判断**(见 §0 判断权铁律)。
4. **验证链自检**:生成后让 DeepSeek 自己核一遍"证据在不在给定材料里"(量自由,这道值得上)。
5. **图谱当校验器**:关系类输出用175只票图谱校验,对不上标存疑。
6. **旁路非阻塞**:报告没生成出来,看板照常能看(学 timsun "AI摘要生成中"占位)。

## 7. 开工顺序(严格按依赖,每步产出可独立用的东西)

> 完整顺序含持仓/监控/推送三个补充环节(详见 SUPPLEMENT_2 §更新)。

- **第0步 · 数据底座**:阿里云跑通 Tushare(用使用者 token)+ 冷启动回灌1年历史 + 验证东财/开源打板数据可采 + yfinance 美股可采。**这步不出成果但是所有模块的粮仓,不通则后面全是空中楼阁。**
- **第0.5步 · 持仓/自选层**:建 holdings/watchlist 表+维护页。**早做**,后续模块都要读它做优先高亮。
- **第1步 · 新闻/事件**:每日报告的原料。集成"仅看我的票"+持仓高亮。做完即有"按链分组事件流+情绪温度计",可独立用。
- **第2步 · 每日报告**:接 DeepSeek 生成盘前盘后+账本,集成"我的持仓动态"板块。心脏跳起来。**正式建之前先拿几条真实新闻实测 DeepSeek 摘要/分类质量 `[待实测]`,别直接信。**
- **第2.5步 · 系统监控 + 基础数据校验**:报告能生成后就加,保护每日依赖(见 SUPPLEMENT_2 §I §J)。
- **第3步 · AI热力图**:175只票表+行情做四象限,持仓票描边。
- **第3.5步 · 推送**:核心模块跑通后加 Telegram P0/P1 推送(见 SUPPLEMENT_2 §H)。
- **第4步 · 研究库 + 基金信函**:慢速层,自动抓+摘要。
