# B6 节点研判卡(二期)设计说明

> 四期演进计划的第二期(2026-07-03 定案:一期头版化 → **二期 B6 研判卡** → 三期 B7 周度成绩单 → 四期 B8 个股决策层)。
> 定位:个人版工具,铁律从"绝不判断"升级为**"判断必须可追责"**——可以有方向,但每个判断带证据链、证伪条件、时间窗,并进 B7 被记分。

## 分工原则

**统计归代码,综合归 LLM。** z-score/共振分/背离检测全部由 `src/research_view/evidence.py` 代码计算;DeepSeek 只做跨源综合与研判措辞(方向/置信/一句话研判/证据链/条件式情景)。LLM 拿到的矩阵与落库的矩阵是同一份快照——防"LLM 自算指标"幻觉,也让 B7 归因有据可查。

## 六源证据矩阵

每交易日盘后,对 48 节点各算六源指标,并在**节点截面**上取 z-score(截断 ±3):

| 源 | 原始值 | 方向性 | 说明 |
|---|---|---|---|
| 新闻 | 当日利好数 − 利空数 | ✓ | raw_news 按 matched_node_ids 聚合 |
| 资金 | 主力 5 日累计净额(亿) | ✓ | moneyflow.multi_day(),EOD 权威口径 |
| 行情 | 节点近一周涨幅 % | ✓ | heatmap_node.ret_1w |
| 龙虎榜 | 上榜票净买额合计(亿) | ✓ | md.top_list.net_amount 最新交易日 |
| 研报 | 近 3 日覆盖篇数 | ✗(注意力) | research_report.node_ids,只计激活不计方向 |
| 信函 | 命中(近45日 relevance≥5 信函文本含节点/链名) | 立场号 ±1 | fund_letter.stance 关键词判 |

**为什么用截面 z**:方向判断的口径本来就是"相对全池超额"(B7 记分口径),截面标准分天然消解主力口径结构性净流出这类系统性偏移——资金 raw −3.9 亿在全池皆负的日子里 z 可以为正,这是特性不是 bug。

## 代码算的三个量

- **共振分** = Σ 方向源权重×z + 信函权重×立场号。权重:行情 1.0 / 资金 1.0 / 新闻 0.8 / 龙虎榜 0.6 / 信函 0.5(龙虎榜稀疏噪音大、信函是低频立场,权重递减)。
- **n_agree / n_active**:与共振方向一致的激活方向源数 / 激活源总数(|z|≥1 或信函命中)。
- **背离**:方向源两两 |z|≥1 且反向 → 客观标注(如"资金×行情反向"),研判卡里 LLM 必须正面处理(说明信哪一源、为什么)。

## 发卡规则

- 候选 = `n_active ≥ 2` **且 scorable**(节点在 heatmap_node 里有 A 股行情数据),按 |共振分| 取 Top 8。
- **scorable 是追责底线**:纯港股映射节点(如整机/本体=优必选)行情永远为 0,B7 记不了分 → 不发卡。记不了分的判断=不可追责。
- 六源矛盾或激活源都弱 → direction 必须"中性"且 thesis 点明矛盾(prompt 硬约束,实测有效:电子特气-气械一体 周涨 +37.7% 但资金连续流出 → 正确判中性)。
- confidence 与同向源数/z 强度匹配,单源驱动不给"高"。

## 存储与消费

- `sql/021_judgment_card.sql`:**append-only**(复用 sql/006 的 ledger_append_only 触发器)。重跑同日只追加,消费方取每节点最新 card_id。卡带完整 matrix/evidence/scenarios/divergence 快照 + model 字段(追责溯源)。
- 编排:`run_pipeline.py` 盘后档 `judgment_cards` 步(hotspots 之后)——须在 EOD 资金/龙虎榜/研报落地后跑,22:30 盘后满足。盘中不发卡。
- 导出:`export.build_dashboard` 挂 `dash.judgment = {date, cards[], fallback}`(当日无回退最近一日并标 fallback)。
- 前端:报告页第 2 屏「节点研判 · 六源共振」(Report.tsx JudgmentCards,A股 only),方向徽章红多绿空 + z 矩阵 chips + 证据链 + 情景/证伪,MoreList 先显 3 张。

## B7 接口(三期要用的都在卡里)

- 打分:direction vs `trade_date + horizon_days(5)` 内该节点**相对全池**超额收益(相对收益排 beta)。
- 归因:错误卡回看 matrix 快照——哪个源的 z 误导了方向(如"新闻 z+2 但资金 z−1.5,错在信了新闻")。
- 校准回路:错误摘要回灌下周 prompt。
- 周日 cron,命中率曲线对错都晒。

## 已知边界

- 信函命中是链级文本匹配(GS 人形机器人信 → 所有机器人节点都命中),偏宽但诚实;立场号只取首个命中信函。
- 龙虎榜/研报是稀疏源,z 截断 ±3 防单源打爆共振分。
- falsify 措辞偶被 LLM 写反(写成"判断成立的条件"),2026-07-03 已收紧 prompt;B7 按实际超额记分,不依赖 falsify 文本。
