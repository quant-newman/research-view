## 三层漏斗收口

候选**只从当日 B6 方向卡(偏多/偏空)节点的成分股里出**——大盘(仪表)→板块(节点卡)→个股(决策卡),没有板块方向支撑的个股不单独发卡。中性节点卡不产生候选。

## 个股六源与对齐分(代码算)

个股在**核心池全体截面**取 z(复用 evidence._z,截断±3):

| 源 | 原始值 | 说明 |
|---|---|---|
| 行情 | 周涨幅 %(heatmap_stock.ret_1w) | 方向源 |
| 资金 | 5日主力净额/总市值(强度,消megacap偏置) | 方向源(md.moneyflow) |
| 新闻 | 当日利好−利空(matched_codes) | 方向源 |
| 龙虎榜 | 净买额(md.top_list.net_amount) | 方向源 |
| 研报 | 近3日覆盖篇数 | 注意力(只加不减) |
| 人气榜 | md.hot_rank(ths源,每日100名,101−rank) | 注意力(P1欠账,至此用上) |

**对齐分** = 节点方向号 × Σ(方向源权重×z) + 0.3×(研报正z + 人气正z)。门槛 ≥1.0,每节点 Top3,全日 ≤12(一次 LLM 调用)。无发卡日收盘价锚(停牌等)不发卡——给不了价位、记不了分。

## 决策卡(DeepSeek)

方向/置信/thesis/**入场/退出/证伪**。硬约束:
- 价位只允许以给出的现价按百分比换算并写明(如"现价488.0的-3%≈473.36"),禁止编造均线/前高等输入外参照;
- 个股证据与节点方向矛盾 → 中性(=放弃候选)并说明——**放弃也落卡也记分**,漏杀好票同样要被看见;
- 偏空卡 = 回避/减仓提示(A股无做空),无入场、exit=解除回避条件;
- falsify = 让方向判断作废的条件,5交易日内可验证;
- B7 lessons 与节点卡共用同一回灌通道(校准期冻结只落库不注入,CALIBRATION_FREEZE,DECISIONS #28)。

实测 2026-07-03:12 卡,金宏气体因"当日-16.2% 与节点偏多矛盾"被正确判中性放弃。

## 存储与追责链

`sql/023 decision_card`:append-only(同 judgment_card),带 **node_card_id**(上游节点卡)——B7 归因可穿透:个股卡错了,能查它依据的板块判断当时怎么说。`close` 列 = 发卡日收盘,entry/exit 的锚。矩阵快照/alignment/model 随卡入库。

## 记分与成绩单(与节点卡同引擎)

- scorecard.score_mature 统一窗口循环:节点卡→card_score(成分等权),个股卡→decision_score(个股收益),同一 verdict 规则(方向卡±1pp外定对错带内平,中性±2pp带内对),同窗口共享一次行情查询;个股停牌无区间行情→pending。
- dashboard_block 加 `stock` 子块(待记分/累计/最近);weekly stats 加 stock_cum/stock_week,错误归纳双层混排(块标【节点】/【个股】,review 键=target)。
