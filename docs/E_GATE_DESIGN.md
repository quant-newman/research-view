## 1. 现状结构事实(代码核对,2026-07-14)

  溯源=事后人工回定位 raw_news/matrix(07-11 抽验 A/B/C/D 法,C-1 即此法产物)。
- B8 entry/exit/falsify 均自由文本;价位规约(D 类)是纸面规则,无机器校验;

- **拒卡机制现状为零**:LLM 输出解析失败只丢字段降级;direction 非法静默归"中性"

## 2. fact_id 与 B1 回指(结构锚)

- **输入块编号化**:发卡输入的每条事实性条目带确定性 id,LLM 在 evidence 中必须回指。
  id 文法(建议,【人拍】):`{src}:{锚}`——
  新闻 `news:<raw_news行id>`(B1 结构化产物的库内主键=B1 回指);
  矩阵类 `mx:<字段路径>`(如 `mx:price.ret_1w`,锚到落库 matrix 同一份);
  研报 `rsch:<行id>`;人气 `hot:rank`;信函 `letter:<行id>`。
- **入库校验(代码,非 LLM)**:evidence 每条 fact_id ∈ 当日该节点/该股输入块 id 集合;
  src 与 id 前缀一致;数字与所指条目一致性属抽验层(哨兵/审计),入库层只校指针有效性。
- prompt 变更:输入块渲染带 id,evidence 输出格式加 `fact_id` 字段——SYSTEM/_USER_TMPL 变更,
  进新 prompt_hash 纪元(§8)。
- 存量卡零触碰:fact_id 只约束新纪元卡;存量 evidence 不重构不回填(卡表 append-only)。

## 3. 证据三分 fact / derived / inference(计划令输入一)

evidence 条目增加 `kind` 字段,三分:

| kind | 定义 | 结构要求 | 校验 |
|---|---|---|---|
| fact | 输入块原文事实 | 必带 fact_id;数字与单位**原样照抄** | 指针有效性(入库);数字一致性(抽验) |
| derived | 由 fact 规约算术得出 | 必带所据 fact_id + **算式全文**(如 `488×0.95=463.6`) | 算式机器复核,误差超容差拒卡 |
| inference | LLM 跨源推断 | 禁带 fact_id(不得伪装成事实);**禁含输入块以外的新数字** | 数字白名单校验(其数字须出现于所引 fact/derived) |

- 禁换算铁律的结构落点:凡换算数字必须以 derived 形态出现并带算式——
  换算数字混在 fact/inference 里 = 拒卡。C-1(463.6 出现在证伪行无算式标注)在本闸下无法入库。
- 与 07-11 抽验分级的映射:A/B≈fact,D≈derived(算式复核通过),C≈derived 缺算式或复核失败→拒卡。
  抽验人工分级自此有结构对应物,抽验转为对闸的复核。

## 4. 必填闸按卡型拆分(缺失一律拒卡记因)

| 卡型 | 必填 | 可空 |
|---|---|---|
| B6 偏多/偏空 | evidence ≥2 条(**至少 1 条 kind=fact**)+ scenarios ≥1 + 证伪条件(具体,horizon 内可验证) | — |
| B6 中性 | thesis 点明矛盾所在 + evidence 仍须全数过 §2/§3 校验 | scenarios 证伪价位行 |
| B8 偏多 | entry(带价位行)+ exit(含止损价位锚)+ falsify + evidence ≥2(≥1 fact) | — |
| B8 偏空 | 回避解除条件(exit 位)+ falsify + evidence ≥2(≥1 fact) | entry |
| B8 中性 | thesis 说明放弃理由 + falsify(放弃错误的可验证信号,【人拍】:现行 prompt 允许中性弱化,是否升必填) | entry/exit |

- **拒卡 ≠ 批次失败**:单卡拒收不阻塞当日批(与 g 件 fail-hard 分层——g 管步骤级,e 管卡级);
  全批被拒 = 诚实空态(卡数 0 + 拒因披露),export 照跑,前端呈现空态横幅(g 件设计稿协同,本稿只定卡级)。
- **记因载体(【人拍】,建议方案 A)**:A=新表 rejected_card(trade_date, card_kind, code/node_id,
  reject_reasons jsonb, raw_llm_json, prompt_hash, created_at;insert-only),可统计入周报;
  B=仅审计目录 JSON 落档(轻但不可 SQL 对账)。建议 A:拒因要进周报披露与对账等式(§7)。

## 5. 价位行机器可验格式(覆盖 entry/exit/falsify 三处)

- 凡含价位的行,结构:`现价 {close} 的 {±N%} → {price}`(或等价结构化子字段,【人拍】文法);
  校验:`|price − close×(1±N%)| ≤ 舍入容差`(容差=价位小数位半步长,同 a 稿误差带方法论)。
- close 锚 = 发卡日收盘(decision_card.close 列,既有)。三处同一文法同一校验函数,单点实现。
- 复核失败 = 拒卡记因(reason=price_formula_mismatch,含三元组 close/N/price 取证)。

## 6. 外生催化标 + 独立事实计数(计划令输入二)

- **外生催化标**:kind=fact 条目可带 `exogenous: true`(政策/订单/业绩预告/停复牌等非行情内生事件)。
  LLM 标注属推断,标注规则文本带版本(exo_rule_v1),错标不拒卡、只作归因维度;
  周报按 exogenous 切片供"纯运气/外生事件"归因(07-12 更正令"纯运气三条标 LLM 推断未经审计"的结构化回应)。
- **独立事实计数 n_independent_facts**:按 event/claim 聚类去重——同一事件多篇报道/多条目=1;
  **禁以 fact_id 条数或文章数冒充**。聚类规则带版本(cluster_v1,判据建议【人拍】:
  同股票/节点 × 同事件类型 × 发布时间 ≤48h × 关键实体重合),纯代码实现,零 LLM。
  **无法可靠聚类时只披露来源条数并如实标注"未聚类"**,不输出独立计数——宁缺不假。
- 用途边界:本件只落计数与标注,不改共振分/对齐分权重,不改候选门槛
  (捆绑第 3 件 n_directional_active≥2 将来若要用独立计数增强,另行登记,本稿不带)。

## 7. 披露与对账(每日/周报)

- 对账等式:**n_候选(或节点数) = n_入库卡 + n_拒卡 + n_LLM漏答**,三项计数随发卡批落
  (与 a 稿 §6、SENTINEL 第 3 项同构:凡不进分母的卡必须显式计数)。
- 周报披露:拒卡数+拒因分布+全批空态日;禁静默吞卡——今日 LLM 漏答某候选即计入 n_LLM漏答。

## 9. 三条必失败测试用例(预注册;实施前必须为红,闸上线后转绿=闸生效证明)

1. **幽灵指针**:evidence 引用当日输入块不存在的 fact_id(如 `news:999999`)→ 拒卡,
