## 六条裁决

1. **样本域**:仅 verdict ∈ {对,错} 的到期卡进入 Brier 与校准统计;verdict=平 剔除,
   但每期必须报告未判定率 n_flat/(n+n_flat)(防 ±1pp 阈值成为事后调节 Brier 样本的
   旋钮)。subjective_prob 为 NULL 的卡(存量旧卡)跳过本统计,主记分照常。
2. **outcome 定义**:对=1,错=0。"判断兑现"完全继承主记分判定(方向卡 ±1pp 阈 /
   中性卡 ±2pp 带),不另设第二套兑现口径。
3. **卡型标记**:每个校准数据点携带 card_kind ∈ {direction, neutral} 与
   card_type ∈ {node, stock},存储时打标,报告时可分层;中性卡与方向卡的基准兑现率
   结构不同,**禁止混桶下结论**。
4. **分桶**:固定边界 [0,0.5), [0.5,0.6), [0.6,0.7), [0.7,0.8), [0.8,0.9), [0.9,1.0),
   自本文档落档日起写死;禁止分位数分桶,禁止任何形式的事后改桶(分位数桶随样本漂移,
   是校准曲线最常见的自欺后门)。
5. **复算规则(审计口径)**:Brier = mean((prob − outcome)²),任何人任何时点抽卡手算
   须与系统累积值一致(对应提案C验收:某张到期卡的 Brier 更新可人工复算一致)。
6. **预期声明**:头几十个数据点的校准曲线预期难看(LLM 已知系统性过度自信,扎堆
   0.7–0.9)。曲线难看=仪器在工作,**禁止以此为由修改 prompt 引导措辞**——那是对
   测量对象的污染,不是修理。

## 附注(读法说明,不改裁决)

- **剔平条件化偏差,读曲线必须配合未判定率**:发卡 prompt 要求模型报的是**无条件**兑现
  概率(带内=未兑现,措辞明示);第 1 条剔平后,桶内实际兑现率对照的是"分出对错条件下"
  的频率,系统性高于无条件率约 1/(1−flat_rate) 倍。故:flat_rate 低时曲线可直读;
  flat_rate 高时先按上式心算去偏,再谈过度/不足自信。这正是第 1 条强制报告未判定率的
  原因——n_flat 不是装饰,是曲线的读数配套件。
- **实现映射**(scorecard.py):`brier_stats`(第1/2/4/5条)、`calibration_block`
  (第3条 card_kind 分层;card_type 由 calibration/stock_calibration 分表天然隔离)、
  `_BIN_EDGES`(第4条边界常量)、`brier_by_version`(#28 版本纪律,跨卡型标量仅作
  漂移检测,不当校准曲线读)。落点:`b7_weekly.stats.calibration / stock_calibration`。
- **版本纪元**:07-06 为复合版本硬边界(prob 自报 prompt 变更 + 参照层 v3 扩容合并为
  单一纪元,#41),prompt_hash 真值 B6 `8528ca795ca4c6b8` / B8 `780916554dc9be8b`;
  效应归因以版本分组整体隔离,禁止事后声称拆出单项贡献。
- 本文只管"概率校准"这一块;主记分口径(命中率/Wilson/基线对照/覆写切片)见
  docs/B7_SCORECARD.md,不受本文影响。
