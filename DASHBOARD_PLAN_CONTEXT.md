> 配套文档:本篇讲 **为什么这么设计**(背景/演进/取舍);

## 一、这个平台是什么,为谁做

**一句话:** 一个自用的 A 股 AI 科技产业链投研看板,服务于一位每天看盘、做短中线决策的独立量化/价值投资者(1–3 人 studio,主力开发即使用者本人)。

**它要解决的真实痛点:** 每天海量的公告、新闻、资金异动、外盘变化里,哪些跟"我关注的 AI 科技产业链(约200只A股)"真正相关?哪个环节的资金和叙事今天起了变化?使用者不想每天人肉刷十几个信息源,想要一个"每天5分钟,告诉我该关注什么"的雷达。

**关键定性(决定一切设计):** 这是**决策雷达**,不是**参考图谱**。它的价值不在于"结构多完整",而在于"每天开盘前/收盘后,让使用者比不看它多知道一件对决策有用的事"。因此:**捕捉变化(增量),不陈列状态(存量)。**

---

## 二、为什么是这六个模块,它们不是平级的

使用者最初想要六个模块:研究库、半导体资讯、每日报告、基金信函、AI热力图、新闻事件。但设计过程中发现:**按"决策雷达"标准看,这六个不是平级的,分三层:**

- **核心层(每天必看,是产品心脏):每日报告 + 新闻事件。** 每日报告是判断层(消费一切、输出"该关注什么");新闻事件是它的原料供应商。
- **支撑层(供数,不单独消费):AI热力图(状态背景板) + 半导体资讯(其实是新闻的行业切片,非独立)。**
- **慢速层(低频高价值,自动抓取):研究库 + 基金信函。**

**为什么这个分层重要:** 它决定了开工顺序(先建原料→再建心脏→再建背景板→最后慢速层),也决定了资源投入优先级。Claude Code 若在支撑层/慢速层上过度雕琢,就是本末倒置。

**使用者已确认:六个模块第一版一起上,且有能力维护。** 所以不砍模块,但分层的主次关系要贯穿始终。

## 三、设计演进(关键的几次转向,理解它们能避免走回头路)

这个方案不是一开始就长这样,经过几次重要转向。Claude Code 了解这些,能避免"优化"回已经被否决的方向:

**转向1:从"新闻聚合器"→"判断雷达"。** 最初设想是做个新闻分类聚合展示。但发现纯聚合只会增加信息噪音,与"减少认知负荷"的目标相悖。**否决了"聚合展示",转向"判断+证伪"的雷达。**

**转向2:从"完整产业链图谱"→"够用就冻结"。** 一度想按中证深交所标准建完整的三表图谱(节点/公司/关系边)。但意识到:**建图谱是"数据管理"的舒适区,会消耗大量精力在"结构正确但对下周涨跌无用"的信息上。** 而使用者要的是"变化",不是"结构完整"。**否决了"持续完善图谱",改为"175只票表冻结够用",图谱降为背景底座 + LLM校验器。**

**转向3:判断基调从"严格证伪"→"牛市视角"。** 使用者明确:不要过度严格、实事求是到违背牛市情绪。牛市里主导定价的是叙事+资金,不是财务质量。**所以:扣非从"筛选闸门"降级为"显示标签"(不砍任何票),主维度改为叙事强度+资金流向。唯一保留的纪律是"证伪条件"。**

**转向4:判断权归属明确 —— LLM 不下判断。** 用了 DeepSeek(便宜、量自由)后,一度有让它多生成、多下判断的诱惑。但确立铁律:**模型对A股产业链理解不如使用者,让它下判断只会得到"流畅但平庸、还诱导放松警惕"的结论。** DeepSeek 严格限定在"呈现变化+提取事实+做分类(选择题)",主线判断永远留白给使用者填。

**转向5:账本从"胜率账本"→"判断复盘账本"。** 一度想统计每日判断的胜率。但意识到:① 每日判断是高噪音信号,胜率数字可能测的是运气不是能力;② 一个可美化的胜率分数会诱发自欺。**改为:记录结构化复盘(判断+证据+证伪条件+错在哪一环),产出"错误类型分布"而非胜率分数,定位为学习工具而非审判工具。** 数据不可篡改靠系统强制(只读+时间戳),不靠人的承诺。

## 四、核心设计原则(施工时的判断准绳)

遇到本文档和工程规格没覆盖的取舍,用这些原则判断:

1. **变化 > 状态。** 任何功能,先问"它捕捉的是变化还是状态?"。变化优先。
2. **使用者判断 > 模型判断。** LLM 呈现和分类,人下判断。永远不让模型替使用者做投资决策。
3. **够用即冻结,别陷入完美。** 尤其图谱和慢速层,避免在"结构完整性"上无限投入。
4. **诚实 > 好看。** 账本要能打自己脸;处理不过来就诚实占位("待分类"),不假装全做完。
5. **喂什么 > 用什么。** LLM 输出质量 90% 取决于喂的数据质量,不是模型选择。防幻觉措施不可放松。
6. **渐进可用。** 每一步产出一个能独立使用的东西,不做"全做完才能用"的大爆炸交付。
7. **牛市基调。** 呈现叙事和资金,不做道德判官(不因估值贵/扣非差就否定一只票),决策权全归使用者。

## 五、不闭门造车:每个设计的真实对标来源

本方案主体没有拍脑袋,关键环节都对标了真实的做得最好的产品/标准。这些来源也是 Claude Code 遇到细节疑问时可以回查的参照:

| 设计部分 | 对标来源 | 学到什么 |
|---|---|---|
| 呈现层(四象限/事件流/情绪温度计) | **timsun.net**(个人宏观研究站,全站实扒) | 判断/证据/证伪四段式、"今天只看3件事"的极致收敛、数据源透明标注、四象限热力图 |
| 数据标准(节点/公司/关系边) | **中证深交所官方课题**《资本市场产业链图谱业务标准及数据标准研究》 | 三表模型、8类双向关系边、申万编码、时间维度;印证"纯A股不做海外对标"符合规范 |
| 盘前盘后模板 | **同花顺早盘必读 / 财联社盘后复盘 / 涨停雷达** | 盘前编号清单(事件+数字+来源)、盘后四件套(情绪/指数/板块/连板)、单条事件黄金模板 |
| 数据源架构 | **开源 a-stock-data**(十层40端点) | 免费源覆盖行情/公告/打板/资金/互动易全套;IP限流注意事项 |
| 防幻觉 | **2026 LLM 研究**(HALLUHARD 等) | 金融领域顶级模型幻觉率仍约30%;行内引用grounding、选择题式、验证链自检、图谱校验 |
| 基金信函做法 | **timsun 基金信函页** | 不自建信源清单,接海外对冲基金季度信聚合圈;卡片式展示;处理不过来标"待分类" |
| 市场验证 | **Wind产业链平台 / 知未智能** | 产业链图谱是Wind几万块的收费功能;证明"产业链视角"方向有市场 |

**差异化(这个平台区别于上述所有对标的地方):**
1. **带使用者自己的产业判断颗粒度**(175只票的龙一/龙二分层、chokepoint映射),这是Wind自动生成给不出的。
2. **牛市视角、带态度**,服务于高β弹性捕捉,这是机构产品出于合规不会做的。
3. **外部事件自动翻译到"我的产业链节点"**(如"黄仁勋说HBM紧"→"利好存储链兆易"),这是通用早报没有的。
4. **免费/低成本数据源**(Tushare付费但便宜 + 开源 + yfinance),复刻Wind几万块的能力。

## 六、诚实的风险提醒(Claude Code 和使用者都该知道)

1. **最大未验证假设是 DeepSeek 的实际能力。** 整套计划把它当核心生成引擎,但其对A股产业链的理解和防幻觉表现**未经实测**。建每日报告前必须先拿真实新闻实测。
2. **两地架构(阿里云+AWS台湾)是推理方案,未验证。** 若跨机推送麻烦,退回全阿里云单机合理。
3. **持续运维是真实负担。** 六模块全上、每天喂养、采集不挂、LLM不出错、账本持续记 —— 这对1-3人团队是不小的持续投入。使用者已确认能维护,但施工时应尽量降低运维复杂度(能自动的自动、能静态的静态)。
4. **"每日判断"这件事本身可能测不出alpha。** 这正是判断复盘账本存在的意义 —— 它会诚实告诉使用者答案。使用者已接受:若账本显示每日判断非强项,就把看板更多当信息聚合、少当判断器。
